상용 모델 비교는 크게 세 가지 방법이 있어요.

1. 리더보드/벤치마크 사이트 활용
이미 누군가 비교해놓은 걸 보는 방법:

Artificial Analysis — 성능, 가격, 속도(TTFT, 토큰/초)를 한눈에 비교. 가장 실용적
Chatbot Arena (LMSYS) — 사람들이 직접 블라인드 투표한 ELO 랭킹. 실사용 품질 비교에 가장 신뢰도 높음
Onyx LLM Leaderboard — 추론, 코딩, 수학, 에이전트 등 카테고리별 비교

2. 비교 기준
품질/성능:
├── MMLU, GPQA, HumanEval 등 벤치마크 점수
├── Chatbot Arena ELO (실사용 체감 품질)
└── 특정 도메인 (코딩, 수학, 한국어 등)

비용:
├── 입력 토큰당 가격
├── 출력 토큰당 가격
└── 컨텍스트 윈도우 크기

속도:
├── TTFT (첫 토큰까지 시간)
├── 출력 토큰/초
└── 지연시간 p95

운영:
├── API 안정성 / SLA
├── 데이터 프라이버시 정책
└── 리전 가용성

3. 직접 평가 (가장 확실)
내 유스케이스에 맞는 테스트셋을 만들어서 직접 비교:


1. 평가 데이터셋 준비 (실제 사용할 프롬프트 50~100개)
2. 각 모델 API로 동일 프롬프트 실행
3. 결과를 사람이 블라인드 평가하거나, LLM-as-a-Judge로 자동 평가
4. 비용/속도도 함께 측정
   
도구로는 OpenAI Evals, LangSmith, Promptfoo 같은 걸 쓸 수 있어요.

실무에서는 보통 리더보드로 후보를 2~3개 추리고, 직접 평가로 최종 결정하는 패턴이에요.
